Лабораторная работа №8

Парсинг веб-данных с помощью BeautifulSoup

Введение

Автоматический сбор данных с веб-сайтов (web scraping) — один из ключевых инструментов в арсенале data scientist'а. Когда нужные данные не предоставляются через API, парсинг HTML-страниц позволяет получить структурированную информацию из открытых источников.

В данной работе реализован парсер новостного портала ИТМО с поиском по ключевому запросу и сохранением результатов в CSV-файл.

Цель работы

Разработать скрипт для автоматического сбора новостей с сайта news.itmo.ru по заданному поисковому запросу с использованием библиотек requests и BeautifulSoup, и сохранить результаты в структурированном виде.

Используемые инструменты

Библиотека Назначение

beautifulsoup4 Парсинг и навигация по HTML-документу

Описание парсера

Архитектура скрипта

Скрипт parser.py выполняет следующую последовательность действий:

  1. Формирует URL поисковой страницы с запросом
  2. Отправляет GET-запрос и получает HTML-ответ
  3. Парсит HTML с помощью BeautifulSoup
  4. Извлекает данные о каждой новости из найденных элементов
  5. Сохраняет результат в itmo_news.csv

Код парсера

import requests
from bs4 import BeautifulSoup
import pandas as pd

itmo_news = []

DOMAIN = 'https://news.itmo.ru'
SEARCH_DOMAIN = 'https://news.itmo.ru/ru/search/?search='
query = 'нейротехнологии'

try:
    response = requests.get(SEARCH_DOMAIN + query)
    response.raise_for_status()
    html_content = response.text
    soup = BeautifulSoup(html_content, "html.parser")

    news = soup.find_all('li', class_='weeklyevent')

    for new in news:
        url = new.h4.a["href"]
        title = new.h4.get_text(strip=True)
        try:
            date = new.find_all('p')[-1].text
        except IndexError:
            date = None

        code = url.split('/')[-2]
        itmo_news.append({
            "Date": date,
            "Title": title,
            "URL": DOMAIN + url,
            "Code": code
        })

except requests.exceptions.RequestException as e:
    print(f"Ошибка при запросе {SEARCH_DOMAIN}: {e}")
except Exception as e:
    print(f"Ошибка при парсинге {SEARCH_DOMAIN}: {e}")

df = pd.DataFrame(itmo_news)
df.to_csv('itmo_news.csv', index=False)

Пояснение ключевых элементов

Поисковый URL: запрос формируется простой конкатенацией строк — SEARCH_DOMAIN + query. Для запросов с пробелами в реальных задачах следует использовать urllib.parse.quote(query).

Поиск элементов: soup.find_all('li', class_='weeklyevent') — находит все новостные карточки на странице. Структура HTML-элемента была предварительно изучена через DevTools браузера.

Извлечение данных из каждой карточки: - new.h4.a["href"] — относительный URL новости - new.h4.get_text(strip=True) — заголовок без лишних пробелов - new.find_all('p')[-1].text — дата (последний тег <p> в карточке) - url.split('/')[-2] — числовой код новости из URL (/ru/.../news/14792/ → 14792)

Обработка ошибок: блок try/except разделяет сетевые ошибки (RequestException) и ошибки парсинга. Если дата отсутствует в разметке — устанавливается None.

Особенности, выявленные при парсинге

Отсутствующая дата. Одна из новостей (код 14405 — «ИТМО и Газпром нефть») не содержит даты в разметке. Скрипт корректно обрабатывает этот случай через try/except IndexError, записывая None. В реальном проекте это значение следует заполнить вручную или через отдельный запрос к странице самой новости.

Кодировка запроса. Кириллический запрос «нейротехнологии» передаётся в URL без явного URL-кодирования. Большинство современных серверов принимают такие запросы, однако для надёжности рекомендуется использовать params={'search': query} при вызове requests.get() — библиотека сделает кодирование автоматически.

Пагинация. Текущая реализация собирает только первую страницу результатов поиска. Если новостей больше, чем помещается на одну страницу, необходимо реализовать перебор страниц.

Возможные улучшения

Пагинация — добавить цикл по страницам результатов:

for page in range(1, max_pages + 1):
    url = f"{SEARCH_DOMAIN}{query}&page={page}"

Расширение данных — для каждой новости можно сделать дополнительный запрос и извлечь полный текст статьи, теги, раздел, автора.

URL-кодирование запроса:

from urllib.parse import quote
response = requests.get(SEARCH_DOMAIN + quote(query))

Задержки между запросами — чтобы не перегружать сервер:

import time
time.sleep(1)

Хранение в базе данных — вместо CSV можно использовать SQLite или PostgreSQL для удобного последующего анализа.
